# Review Checklist

Use this checklist before merging or handing work off:

- Does the change alter behavior in a way that needs a note in the README or release notes?
- Are there edge cases around empty input, missing files, or partial configuration?
- Did we preserve existing user changes and local state?
- Is the new code consistent with the surrounding style?
- Did we run the narrowest useful validation step?
- If validation was skipped, did we say exactly what was not checked?
